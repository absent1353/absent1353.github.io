Leetcode之股票问题

动态规划
力扣
发布日期:   2021-04-10

股票问题系列通解

1.包含题目

LeetCode121.买卖股票的最佳时机

LeetCode122.买卖股票的最佳时机 II

LeetCode123.买卖股票的最佳时机 III

LeetCode188.买卖股票的最佳时机 IV

LeetCode309.最佳买卖股票时机含冷冻期

LeetCode714.最佳买卖股票时机含手续费

2. 通用情况

符号介绍：

  • n表示股票价格数组的长度
  • i表示第i天(取值为0~n-1)
  • k 表示允许的最大交易次数
  • 用T[i][k]表示在i天结束时，最多进行k次交易的情况下可以获得的最大收益。

动态规划的基准情况为$T[-1][k] = T[i][0] = 0$，表示没有进行股票交易时没有收益，第一天对应i=0，因此i=-1表示没有股票交易。如果可以将T[i][k]关联到子问题，例如T[i-1][k]，T[i][k-1]，T[i-1][k-1]等子问题，就能得到状态转移方程。如何得到状态转移方程，最直接的办法是看第i天可能的操作，本题型中操作为买入，卖出，休息，通过计算三种操作获取最大收益，当然限制条件为：规定不能同时进行多次交易，因此如果决定在第i天买入，在买入之前必须持有0份股票，如果决定在第i天卖出，在卖出之前必须恰好持有1份股票，持有股票的数量是上文提及到的隐藏因素，该因素是影响第i天可以进行的操作，进而影响最大收益。

对T[i][k]的定义需要分成两项：

T[i][k][0] 表示第i天结束时，最多进行k次交易且在进行操作后持有0份股票的情况下获得的最大收益(金钱数)。

T[i][k][1]表示在第i天结束时，最多进行k次交易且在进行操作后持有1份股票的情况下可以获得的最大收益(金钱数)。

使用新的状态表示后，基准和状态转移方程分别为：

基准情况：

T[-1][k][0] = 0
T[-1][k][1] = -Infinity
T[i][0][0] = 0
T[i][0][1] = -Infinity

状态转移方程：

T[i][k][0]=max(T[i - 1][k][0], T[i - 1][k][1] + prices[i])
T[i][k][1] = max(T[i-1][k][1], T[i-1][k-1][0] - prices[i])

基准情况中，$T[-1][k][0] = T[i][0][0] = 0 $的含义与上文同，$T[-1][k][1]= T[i][0][1] = -Infinity$ 含义指在没有进行股票交易时不允许持有股票。

对于状态转移方程中的T[i][k][0]，第i天进行的操作只有休息或卖出，因为在第i天结束时持有的股票数量为0。T[i - 1][k][0] 是休息操作可以获得最大收益，T[i - 1][k][1] + prices[i]是卖出操作可以得到最大收益。允许的最大交易次数是不变的，因为每次交易包含两次成对的操作，买入和卖出。只有买入操作会改变允许的最大交易次数

对于状态转移方程中的T[i][k][1]，第i天进行的操作只有休息或买入，因为在第i天结束时持有股票数量为1。T[i-1][k][1] 是休息操作可以得到的最大收益，T[i-1][k-1][0] - prices[i]是买入操作可以得到的最大收益。允许的最大交易次数减少了一次，因为每次买入操作会使用一次交易。

为得到最后一天结束时最大收益，遍历股票价格数组，根据状态转移方程计算T[i][k][0]和T[i][k][1]的值，最终答案是T[n-1][k][0]，因为结束时持有0份股票收益一定大于持有1份股票的收益。

3. 应用

3.1. k=1

每天有两个未知变量，T[i][1][0]和T[i][1][1]，状态转移方程如下：

T[i][1][0] = max(T[i-1][1][0],T[i-1][1][1]+prices[i])
T[i][1][1] = max(T[i-1][1][1],T[i-1][0][0]-prices[i]) = max(T[i - 1][1][1], -prices[i])

第二个状态转移方程利用了$T[i-1][0][0]=0$

题目： LeetCode121 买卖股票的最佳时机

给定一个数组 prices ，它的第 i 个元素 prices[i] 表示一支给定股票第 i 天的价格。

你只能选择 某一天 买入这只股票，并选择在 未来的某一个不同的日子 卖出该股票。设计一个算法来计算你所能获取的最大利润。

返回你可以从这笔交易中获取的最大利润。如果你不能获取任何利润，返回 0 。

输入：[7,1,5,3,6,4]
输出：5
解释：在第 2 天（股票价格 = 1）的时候买入，在第 5 天（股票价格 = 6）的时候卖出，最大利润 = 6-1 = 5 。
     注意利润不能是 7-1 = 6, 因为卖出价格需要大于买入价格；同时，你不能在买入前卖出股票。

解题思路：

推导状态转移方程：

dp[i][0]：规定了今天不持股，有以下两种情况：

1.昨天不持股，今天什么都不做；
2.昨天持股，今天卖出股票（现金数增加），
dp[i][1]：规定了今天持股，有以下两种情况：

  1. 昨天持股，今天什么都不做（现金数与昨天一样）；
  2. 昨天不持股，今天买入股票（注意：只允许交易一次，因此手上的现金数就是当天的股价的相反数）
class Solution {
    public int maxProfit(int[] prices) {
        int len = prices.length;
        // 特殊判断
        if (len < 2) {
            return 0;
        }
        int[][] dp = new int[len][2];
        // dp[i][0] 下标为 i 这天结束的时候，不持股，手上拥有的现金数
        // dp[i][1] 下标为 i 这天结束的时候，持股，手上拥有的现金数
        // 初始化：不持股显然为 0，持股就需要减去第 1 天（下标为 0）的股价
        dp[0][0] = 0;
        dp[0][1] = -prices[0];
        // 从第 2 天开始遍历
        for (int i = 1; i < len; i++) {
            dp[i][0] = Math.max(dp[i - 1][0], dp[i - 1][1] + prices[i]);
            // 这里采用dp[i][1] = -prices[i]是因为只允许交易一次，因此在买(-prices[i])之前是0，而不是dp[i][0]
            dp[i][1] = Math.max(dp[i - 1][1], -prices[i]);
        }
        return dp[len - 1][0];
    }
}

思路2：

class Solution {
    public int maxProfit(int[] prices) {
        int len = prices.length;
        // 特殊判断
        if (len < 2) {
            return 0;
        }
        int[] dp = new int[len];
        dp[0] = 0;
        int minprice = prices[0];
        for(int i=1;i<len;i++){
            minprice = Math.min(minprice,prices[i]);
            dp[i] = Math.max(dp[i-1],prices[i]-minprice);
        }
        return dp[len-1];
    }

}

更优化一步：

空间优化，只看状态转移方程，状态转移方程下标为i的行只参考下标为i-1的行，并且：

  • 下标为i的行并且状态为0的行参考了上一行状态为0和1的行
  • 下表为i 的行并且状态为1的行只参考了上一行的状态为1的行
class Solution {
    public int maxProfit(int[] prices) {
        int len = prices.length;
        // 特殊判断
        if (len < 2) {
            return 0;
        }
        int[] dp = new int[2];
        dp[0] = 0;
        dp[1] = -prices[0];
        for(int i=1;i<len;i++){
            dp[0]=Math.max(dp[0],dp[1]+prices[i]);
            dp[1]=Math.max(dp[1],-prices[i]);
        }
        return dp[0];
    }

}
